This is a live systems language project which aims to append custom DSLs and algebraic structure onto hardware.

> [!WARNING]
> This project is in a sketch phase so intense that it might be a sketch in the comedy sense. If you're here, I hope it is to laugh with me.

This README will be equal parts manifesto, opinions, and features to work towards.  

## Aches And Painkillers

Since I started programming, hardware has been the most frustrating part.  
As soon as you start pulling back the veil of what information really is and where it really lives, shit starts feeling too esoteric, ugly, and sterile (to me).  
I would like to append algebraic structures onto hardware just to make it make sense to me. I'm a mathematician. Not an electronics engineer, nor a traditional CS person. I want actions and maps that help me to feel what hardware does and how it can be defined as a live entity. I feel like if I allow hardware to speak its own language, I will understand it better.
